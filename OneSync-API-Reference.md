## What it is

OneSync is a **ClassLink** identity/roster sync product.

- **`wwwroot/cl_onesync/`** — **Angular** SPA (PrimeNG UI, PWA service worker). The frontend.
- **`wwwroot/cl_onesync/v1p0/`** — **ASP.NET Core 8** API (`OneSync.dll`) + domain logic (`OneSyncLibrary.dll`), hosted under IIS (`AspNetCoreModuleV2`, out-of-process).
- **Background workers**, each a separate service/exe: `CronWorker`, `QueueWorker`, `ImportWorker`, `CsvWriterWorker`, `MaintenanceWorker`, `MonitorWorker`, `OmsWorker`, `SystemNotificationsWorker`. They run imports/exports/CSV jobs off a **RabbitMQ** queue. Notably, each can be toggled "in-process" via a `settings/property/start<Name>WorkerInProcess` flag.

Integrations (from bundled SDKs): Google Admin Directory / Drive / Sheets / Gmail, Microsoft Graph (Entra/Azure AD), AWS S3, FTP/SFTP, ODBC, CSV, and ClassLink directory/OneRoster.

## How the client talks to the API (verified)

```
baseUrl   = window.location.origin + "/v1p0"      // dev: http://localhost:5000
auth      = Authorization: Bearer <token>          // token from localStorage "token"
headers   = Content-Type: application/json         // text/csv for CSV download endpoints
```

- A JWT is obtained at login and stored in `localStorage.token`; the API service appends it to every request.
- CSV exports set `Content-Type: text/csv` and stream a file; the client builds a timestamped filename client-side.
- `auth/status` is called with a `?page=` style discriminator (`login` vs `dashboard`).
- "whats-new"/release content is pulled from an external host: `https://whatsnew.apis.classlink.com/whatsnew/v1p0/products/{id}` (Bearer-auth), separate from the OneSync API.

All paths below are relative to `/v1p0`.

## Logs — `logs`

## Settings — `settings`

Two big generic patterns plus many specific endpoints.

**Generic key/value:** `GET settings/property/{key}` and `PUT settings/property/{key}`. Observed keys cluster into:

- **Retention** (each with `Verbose`/`Debug` siblings): `importRetention`, `exportRetention`, `systemRetention`, `workerRetention`, `dryRetention`, `correlationRetention`, `workflowsRetention`.
- **Email toggles/schedules:** `importEmailsSuccessesEnabled`, `importEmailsFailuresEnabled`, `importEmailsThresholdEnabled`, `importEmailsThresholdsOverriddenEnabled`, `exportEmailsEnabled`, `exportEmailsOnFailOnly`, `exportEmailsSchedule`, `destinationThresholdsEmailEnabled`, `systemEmailList`, `MailServer`.
- **Export batching:** `exportBatchAutomatic`, `exportBatchSyncAll(Dry)`, `exportBatchSyncChanges`, `exportBatchCollection`, `exportBatchBulkUserList`, `exportBatchIndividualUserList(Dry)`.
- **Worker process flags:** `startCronWorkerInProcess`, `startQueueWorkerInProcess`, `startImportWorkerInProcess`, `startCsvWorkerInProcess`, `startMaintenanceWorkerInProcess`, `startMonitorWorkerInProcess`, `startOmsWorkerInProcess`, `startSystemNotificationsWorkerInProcess`, plus `useBackgroundWorkers`, `jobQueueProcessing`, `workerQuantity` (fast/slow/normal), `concurrentImportLimit`, `importTimeout`.
- **Remote-access email templates:** `remoteAccessRequestedEmail`, `remoteAccessApprovedEmail`, `remoteAccessRejectedEmail`, `remoteAccessRevokedEmail`.
- **System/misc:** `versionNum`, `logLevel`, `language`, `oneSyncEnvironment`, `oneSyncHeaders`, `optInBeta`, `customYOGValues`, `yearRollover`, `rmq_code`, `rmq_error`.

**Per-user preferences:** `GET/PUT settings/userPreference/{key}` — `darkTheme`, `themeColor`, `language`, `humor`, `growlLife`, `growlRegion`, `showAdvancedFilters`, `useAnchorTabs`. Plus `GET settings/userPreference` (all).
